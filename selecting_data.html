SELECTING DATA

MULTIPLE COLUMNS

To select all columns in the table, use an * instead of naming the column

When selecting columns from multiple tables, use the fully qualified name: TableName.ColumnName

To select multiple specific columns, use commas to list them all (except after the last column)

SELECT customer.id, customer.experience FROM customers;



UNIQUE RECORDS

In situations in which you have multiple duplicate records in a table

It might make more sense to return only unique records, instead of fetching the duplicates

The DISTINCT keyword is used in conjunction with SELECT to eliminate all duplicate records and return only unique ones

SELECT DISTINCT experience FROM customers;



LIMITED RETURN SET

By default, all results that satisfy the conditions specified in the SQL statement are returned.

However you may just need a subset of entries, such as the first five records.

This is done by using the LIMIT keyword

SELECT customers.id FROM customers WHERE customers.experience = 'shopper' LIMIT 5;
